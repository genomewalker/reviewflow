## API Endpoints

| Endpoint | Method | Description |
|----------|--------|-------------|
| `/papers` | POST | Create new paper entry |
| `/papers/{id}/process` | POST | Upload files & start processing |
| `/papers/{id}/status` | GET | Poll processing job status |
| `/papers/{id}/data` | GET | Get full paper data |
| `/papers/{id}` | DELETE | Soft delete paper |
| `/papers/trash` | GET | List deleted papers |
| `/papers/{id}/restore` | POST | Restore from trash |

## Database Schema

```sql
-- Core tables
papers (id, title, authors, journal, field, status, created_at, ...)
reviewers (id, paper_id, name, expertise, sentiment, ...)
comments (id, paper_id, reviewer_id, type, category, priority, original_text, ...)

-- Processing tracking
processing_jobs (id, paper_id, status, progress, current_step, logs, ...)

-- Session management
sessions (id, opencode_session_id, model, agent, variant, paper_id, ...)

-- Expert analysis cache
expert_discussions (stored in JSON file and localStorage)
```
